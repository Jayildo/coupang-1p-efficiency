# Coupang 1P Efficiency Workbench

> 쿠팡 1P(직매입) 셀러의 반복 작업을 한 화면에서 처리하는 브라우저 워크벤치.
> 발주서 정리, 발주서 PDF 분할, 팔레트/박스 적재 시뮬레이션, 사용자 피드백을
> 단일 SPA로 묶었다.

## Problem

쿠팡 1P 셀러는 매일 다음 작업을 반복한다:
- 쿠팡에서 받은 발주 엑셀을 정리해서 SKU 단위로 다시 펼치기
- 여러 페이지 PDF에서 필요한 페이지만 골라 분할/병합
- 받은 발주량을 팔레트와 박스에 어떻게 적재할지 사전 계산
- 동료/거래처에 "이렇게 하면 편한데" 같은 개선 요청을 흘려보내기

각 작업마다 별도 도구를 띄우고, 파일을 로컬에 저장하고, 결과를 다시 합치는
손맛 없는 흐름이 매일 [수십 분] 단위로 누적된다.

## Hypothesis

- 위 4개 작업은 모두 **브라우저 안에서 끝낼 수 있는 작업**이다 (서버 불필요).
- 한 SPA에 묶고 탭만 전환하면, 도구 전환 비용이 0이 된다.
- 피드백 채널을 같은 화면에 박아두면, 평소엔 잊어버리는 개선 요청이
  즉시 기록된다.

## Decision

| 결정 | 채택안 | 버린 안 | 이유 |
|---|---|---|---|
| 백엔드 | 정적 SPA + Supabase (피드백/활성 사용자만) | 자체 서버 | 운영 비용 0, 핵심 로직은 클라이언트에서 충분 |
| PDF 처리 | `pdfjs-dist` + `pdf-lib` 클라이언트 사이드 | 서버 업로드 | 사용자 발주서를 서버에 올리지 않음 (프라이버시) |
| 엑셀 처리 | `xlsx` SheetJS 클라이언트 사이드 | 서버 변환 | 동일 — 데이터 외부 전송 없음 |
| 적재 시뮬 | 팔레트 CBM 1.65 기준 단일패스 계산 | 외부 솔버 | 입점사 평균 SKU 수 기준으로 단순 룰이 충분 |
| 피드백 UX | 카테고리 + 이모지 반응 + 상태 추적 | 단순 텍스트 폼 | "급해요🔥" 표시로 우선순위 자연 정렬 |

## Outcome

- 4개 작업 도구 통합 → **탭 전환 시간 = 0**
- 발주서 PDF가 사용자 PC를 떠나지 않음 → 보안/프라이버시 부담 0
- 피드백 누적 → 다음 버전 우선순위 자동 정렬

## Tradeoffs

- **인증 없음**. 피드백 좋아요/이모지는 익명. 어뷰징 가능성 인지하고 감수.
- 1P 셀러 도메인 한정. 3P/오픈마켓에는 그대로 안 맞음 (적재 룰부터 다름).
- 클라이언트 사이드 처리 → **대용량 PDF에서 느려진다**. 50MB 이상은 분할 권장.

## Modules

- **OrderWorkbench**: 쿠팡 발주 엑셀 → SKU 단위 펼치기 + 재집계
- **DocumentWorkbench**: PDF 다중 파일 업로드 → 페이지 필터링/분할/zip 다운로드
- **LoadingWorkbench**: 팔레트/박스 적재 시뮬레이션 + 인쇄용 출력
- **FeedbackWorkbench**: 카테고리·이모지 반응 기반 사용자 피드백 (Supabase)

## Stack

React 18 · Vite · Supabase (피드백 only) · pdfjs-dist · pdf-lib · xlsx · lucide-react
